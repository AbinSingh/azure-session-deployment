
# How FastAPI Processes This Request
Suppose you send:

POST /login
Content-Type: application/json

{
    "username":"alice",
    "password":"password123"
}

The flow is:

    Browser
          │
          ▼
    HTTP Request
          │
          ▼
    Uvicorn
          │
          ▼
    ASGI Scope
          │
          ▼
    FastAPI
          │
          ▼
    Pydantic
    
    Creates
    
    LoginRequest(
        username="alice",
        password="password123"
    )
          │
          ▼
    login()
          │
          ▼
    authenticate()
          │
          ▼
    True
          │
          ▼
    JSON Response

Notice something new:

FastAPI automatically creates a LoginRequest object from the JSON body.
You didn't write any JSON parsing code.

# Observe the HTTP Response
For a successful login, the response headers will look similar to:

# Response Body
{
    "message":"Authentication successful",
    "username":"alice"
}
# Response Header:
HTTP/1.1 200 OK
Content-Type: application/json
Content-Length: ...
date: Mon,13 Jul 2026 13:20:28 GMT 
server: uvicorn 


Notice what is missing:
    No Set-Cookie header.
    No Cookie header.
    No session identifier.

So if you immediately call another endpoint (for example, a future /profile), the server has no memory of who you are. It would treat that request as coming from a brand-new client.
