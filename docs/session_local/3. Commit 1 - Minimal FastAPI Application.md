# Goal

Before we talk about sessions, cookies, or authentication, we need to understand the smallest possible FastAPI application.

At this stage our application should simply answer one question:

"Can Uvicorn successfully invoke my FastAPI application?"

Nothing more.

No cookies

We're only proving that the request reaches our application.

# What did FastAPI do?

FastAPI's responsibilities were:

Register the route GET /.
Match the incoming request to the health_check() function.
Convert the returned Python dict into JSON.
Produce a 200 OK HTTP response.

-- It did not:

Open a network socket.
Listen on a TCP port.
Parse raw HTTP bytes.
Manage TCP connections.

Those tasks are handled by Uvicorn and the operating system.

# What did Uvicorn do?

Uvicorn's responsibilities were:

-- Open port 8000.
-- Listen for incoming TCP connections.
-- Parse the HTTP request into an ASGI-compatible structure.
-- Invoke the FastAPI application.
-- Receive FastAPI's response.
-- Serialize it into an HTTP response and send it back over the socket.
