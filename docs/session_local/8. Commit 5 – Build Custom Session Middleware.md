
# Middleware

If you understand middleware well, you'll understand not only FastAPI, 
but also Django, Express.js, ASP.NET Core, Spring Boot, and many other 
frameworks—they all have the concept of middleware or filters/interceptors.

# Commit 5 – Build Custom Session Middleware

# Big Picture

So far, the browser automatically sends the cookie:

Cookie: session_id=c57727b5-4eb3-4df6-bca4-cd8d30764fe4

-- But who reads it?
    Not Uvicorn.
    Not the OS.
    Not the browser anymore.
    It is our application's responsibility.
    We don't want every endpoint to repeatedly do this:

cookie = request.cookies.get("session_id")
if cookie:
    session = get_session(cookie)

That would duplicate code everywhere.

Instead, we'll do it once in middleware.

# Architecture Before Commit 5

    Browser
        │
        ▼
    Uvicorn
        │
        ▼
    FastAPI
        │
        ▼
    Route Handler

The route handler would have to parse cookies itself.

# Architecture After Commit 5

    Browser
        │
        ▼
    Uvicorn
        │
        ▼
    Session Middleware
        │
        ▼
    Route Handler

Now every request flows through the middleware first.

# What is Middleware?

Think of middleware as a security checkpoint at an airport.

Passenger
    │
    ▼
Security Check
    │
    ▼
Boarding Gate

Every passenger passes through security.

    Similarly,
    
    HTTP Request
          │
          ▼
    Middleware
          │
          ▼
    Route Handler
    
    Every request passes through middleware.

# What Will Our Middleware Do?

Every incoming request:

Receive Request
      │
      ▼
Read Cookie
      │
      ▼
Extract session_id
      │
      ▼
Lookup Dictionary
      │
      ▼
Attach Session to request.state
      │
      ▼
Call Route Handler
      │
      ▼
Return Response

# Why request.state?

FastAPI's Request object contains a special attribute called state.

Think of it as a backpack attached to the request.

Initially:

    request.state
    
    is empty.

Middleware can add anything:
    request.state.user
    request.state.session
    request.state.request_id
    request.state.start_time
Later, your endpoints simply read these values.
