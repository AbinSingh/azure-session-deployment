# 1.2 Update get_session()

    def get_session(session_id: str):
    
        session = sessions.get(session_id)
    
        if session is None:
            return None
    
        now = datetime.utcnow()
    
        last_accessed = session["last_accessed"]
    
        if now - last_accessed > timedelta(minutes=SESSION_TIMEOUT_MINUTES):
    
            delete_session(session_id)
    
            return None
    
        session["last_accessed"] = now
    
        return session

This is the heart of idle timeout.

# 1.3 What Happens?

Suppose 09:00 Login , last_accessed = 09:00

User visits profile , 09:10 , Update last_accessed = 09:10

Another request , 09:20 , Update last_accessed = 09:20

No activity. Time becomes , 09:55 , Difference 35 minutes

Greater than 30 minutes , Middleware receives None

User is effectively logged out.

# 1.4 No Middleware Changes!

This is beautiful. Middleware still does

session = get_session(session_id)

The middleware doesn't need to know why None was returned.

Maybe:
    invalid session
    deleted session
    expired session

Middleware simply treats them all the same.

That's excellent software design.

# Why Redis is Better

Now you can appreciate why Redis exists.

Our current code

    if expired:
    delete_session()
In Redis
    SET session:abc123 ... EX 1800

Redis automatically removes the key after 1800 seconds.

No cleanup code.

No background thread.

No memory leak.

The infrastructure handles expiration.

This is one reason Redis is such a good fit for session storage.
