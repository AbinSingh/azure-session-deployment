# What is ASGI?

Think of ASGI as a standard interface. Without ASGI, every framework would expect requests in a different format.

Instead, every ASGI server (Uvicorn, Hypercorn, Daphne, etc.) creates the same Python object.

That object is called the scope.

# ASGI Scope

scope = {
    "type": "http",
    "http_version": "1.1",
    "method": "GET",
    "scheme": "http",
    "path": "/",
    "query_string": b"",
    "client": ("127.0.0.1", 51584),
    "server": ("127.0.0.1", 8000),
    "headers": [
        (b"host", b"localhost:8000"),
        (b"user-agent", b"Mozilla/5.0"),
        (b"accept", b"text/html"),
    ]
}

# Think of the scope as a translation
Browser sends GET /
↓
Uvicorn converts it into
{
    "method": "GET",
    "path": "/"
}
FastAPI never sees the raw HTTP text. It only sees this structured Python data.

# Where do cookies fit?

In Commit 1, the HTTP response looks like this: step 7

There is no Set-Cookie header. So the browser stores nothing.

Later, in the login endpoint, the response will look like:

    HTTP/1.1 200 OK
    Content-Type: application/json
    Set-Cookie: session_id=abc123xyz; HttpOnly; Secure
    
    {
        "message":"Login successful"
    }

The only difference is the additional Set-Cookie response header.

The browser sees that header and saves the cookie automatically. On subsequent requests to the same domain/path, it includes:

Cookie: session_id=abc123xyz

That's the entire session mechanism from the browser's perspective.

# Where will middleware fit?

Right now, the flow is:
    Browser
        │
        ▼
    Uvicorn
        │
        ▼
    FastAPI
        │
        ▼
    Route Handler

After we introduce middleware, the flow becomes:

    Browser
        │
        ▼
    Uvicorn
        │
        ▼
    Session Middleware
        │
        ▼
    FastAPI Route

The middleware gets the first opportunity to inspect every request. It will:

Read the Cookie header.
Extract session_id.
Look up the session in our in-memory dictionary.
Attach the session to request.state.
Pass the request to the route handler.

Because this happens before your endpoint runs, every protected endpoint can simply read request.state.session instead of repeating cookie parsing and session lookup logic.

# One small correction to your earlier mental model

Earlier you said: "Uvicorn is an ASGI."

A more precise statement is:

ASGI is a specification (an interface/contract).
Uvicorn is an ASGI server that implements that specification.
FastAPI is an ASGI application that follows that specification.

So the interaction is:

    HTTP Request
          │
          ▼
    Uvicorn (ASGI Server)
          │
          │ Calls ASGI interface
          ▼
    FastAPI (ASGI Application)
          │
          ▼
    Your endpoint function

This distinction becomes especially useful later in AKS, because you can swap Uvicorn for another ASGI server (such as Hypercorn) without changing your FastAPI application—the contract between them remains the ASGI specification.
