# What is Authentication?

Authentication answers only one question:

"Who are you?"

Example:
    Username : alice
    Password : password123

Server checks:
    Is this username/password valid?

If yes:
    200 OK

If no:
    401 Unauthorized

That's all. The server does not remember you.

# Authentication vs Session

This distinction is very important.

# Authentication
    Client
       │
       ▼
    Username + Password
       │
       ▼
    Server
       │
       ▼
    Valid?

# Session
    Login Once
          │
          ▼
    Generate Session ID
          │
          ▼
    Store Session
          │
          ▼
    Remember User

Session comes after successful authentication.

from pydantic import BaseModel

class LoginRequest(BaseModel):
    username: str
    password: str

# Why?
When the browser sends:
    {
        "username": "alice",
        "password": "password123"
    }
    FastAPI automatically converts it into a Python object:
        login.username
        login.password
    instead of making you parse JSON manually.

# Step 3 — auth.py

This file contains our authentication logic.

VALID_USERS = {
    "alice": "password123",
    "bob": "password123"
}
def authenticate(username: str, password: str) -> bool:
    """
    Validate username and password.
    """
    if username not in VALID_USERS:
        return False
    return VALID_USERS[username] == password

# Why separate auth.py?

Enterprise applications rarely put authentication inside route handlers.

Instead:
    Route
       │
       ▼
    Authentication Service
       │
       ▼
    Database / LDAP / Azure AD

Today our "database" is just a dictionary.

Later we can replace it without changing the API.
