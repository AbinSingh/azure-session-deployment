The browser treated this as ordinary JSON.

Browsers only do that when they receive a Set-Cookie HTTP response header.

# What is a Cookie?

A cookie is simply a small key-value pair stored by the browser.

Example: session_id = c57727b5-4eb3... That's it. The browser stores it.

Later, the browser automatically sends Cookie: session_id=c57727b5-4eb3...

You never write JavaScript to do this. The browser does it.

# Step 1 - Update app.py

We need FastAPI's Response object because that's how we modify HTTP headers.

# What does response.set_cookie() do?

It adds this HTTP header:
Set-Cookie:
session_id=c57727b5...

FastAPI hides the low-level HTTP formatting.

Internally it's just creating another response header.

FastAPI merges the cookies from the injected Response into the JSON response.

# Why HttpOnly=True?

This is a security feature.  Without it, JavaScript can read the cookie.

With HTTPonly, JavaScript cannot access the cookie.

Only the browser sends it with HTTP requests. This helps mitigate certain attacks, such as XSS stealing session cookies.
