Before Commit 3

Commit 3 is where the application becomes stateful.

We'll introduce three new concepts together:
    Session IDs — why they must be random and unguessable.
    Session Store — our in-memory dictionary that maps a session ID to user data.
    Cookie Creation — how FastAPI adds a Set-Cookie header so the browser automatically stores the session ID.
This is the point where you'll see the transition from stateless authentication (every request stands alone) to stateful sessions (the server can recognize subsequent requests from the same browser).
