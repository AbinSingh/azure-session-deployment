This is also a good time to point out something important about enterprise software design.

Business logic should not care where authentication came from.

Your /profile endpoint shouldn't know anything about:
    cookies,
    HTTP headers,
    session IDs,
    dictionaries,
    Redis.

It should only ask one question:

"Has middleware already identified the user?"

That's a fundamental design principle.

# Commit 6 - Protect /profile

# Big Picture

Until now we have built the following:

    Browser
        │
        │ Cookie: session_id=abc123
        ▼
    Uvicorn
        │
        ▼
    Session Middleware
        │
    Reads Cookie
        │
    Looks up Dictionary
        │
    request.state.session
        ▼
    Route Handler

Notice that middleware has already done all the hard work.

The route handler simply consumes the result.

# Separation of Responsibilities

# Middleware
Responsible for
    Read Cookie
    ↓
    Read Session ID
    ↓
    Lookup Session Store
    ↓
    Populate request.state.session
# Route Handler
Responsible for
    Business Logic
    ↓
    Return User Profile
The route should never parse cookies.

# Current Request Flow

Imagine Alice has already logged in.

The browser automatically sends

GET /profile

Cookie:
session_id=abc123

The flow becomes

    Browser
        │
        ▼
    Uvicorn
        │
        ▼
    Middleware
        │
    Cookie -> abc123
        │
    Dictionary Lookup
        │
    request.state.session =
    {
       "username":"alice"
    }
        │
        ▼
    Profile Endpoint
        │
    Returns Profile
