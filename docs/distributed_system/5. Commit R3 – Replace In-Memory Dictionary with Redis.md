# 1. Big Picture

-- Before
                  FastAPI
                      │
                      ▼
            Python Dictionary (RAM)

Every session lived inside the Python process.

-- After
                  FastAPI
                      │
                 Redis Client
                      │
                 TCP Socket
                      │
                      ▼
                 Redis Server

Now the session lives in another process. This is exactly what enterprise applications do.

# What Changes?

Only one file. session_store.py Everything else stays exactly the same.

That's the beauty of abstraction.

# 4. Step 3 – Update create_session()

    redis_client.set(
        session_id,

    )

-- What happened?
Instead of writing into Python RAM, FastAPI sends a TCP packet to Redis.
Redis stores the value in its own memory.

# 5. Step 4 – Update get_session()

session_json = redis_client.get(session_id)

session = json.loads(session_json)

Redis returns '{"username":"bob"}' which is a string.

Python needs

{
    "username":"bob"
}

Therefore json.loads() converts JSON → dictionary.

# When updating

session["last_accessed"] = now.isoformat()

Python has a dictionary. Redis needs a string.

So we convert back. json.dumps(session) , dictionary → JSON string.

    ↓
    All Sessions Lost

Kill FastAPI
↓
Redis Still Running
↓
Sessions Still Exist

This is the first major enterprise benefit of moving state out of the application.

# 10. Before Commit R4

Commit R4 will be one of the most satisfying demonstrations in the project.

We'll prove that the session truly lives outside FastAPI:

    --Log in.
    --Confirm the session exists in Redis.
    --Stop Uvicorn (Ctrl+C).
    --Start Uvicorn again.
    --Without logging in again, call GET /profile.

If the request still succeeds, you'll have witnessed a fundamental principle of distributed systems:

The application process is stateless; the session state lives in a separate service (Redis).

That's the same architectural pattern you'll later use with Azure Cache for Redis and AKS, just at enterprise scale.
