# 1. Big Picture

Before this commit, the architecture looked like this:

                  Windows
                      │
      ┌───────────────┴───────────────┐
      ▼                               ▼
 python.exe                     redis-server.exe
(FastAPI)                        (Redis Server)

The two processes existed, but they didn't know about each other.

After this commit:

                  Windows
                      │
      ┌───────────────┴───────────────┐
      ▼                               ▼
 python.exe                     redis-server.exe
(FastAPI)                        (Redis Server)
      │                               ▲
      │        TCP Connection          │
      └───────────────────────────────►│
 
FastAPI can now talk to Redis. We're not storing sessions yet.

We're only proving that the connection works.

# 2. Learning Goal

By the end of this commit, you'll understand:

1. What a Redis client library is.
2. How FastAPI establishes a TCP connection to Redis.
3. Why Redis is just another network service.
4. How to verify connectivity before changing application logic.

This is exactly how enterprise teams work. They first verify infrastructure connectivity, then migrate functionality.


# 3. Step 1 – Install the Redis Python Client

    Update requirements.txt

Notice:
We're not installing Redis. Redis is already running.
We're only installing a Python library that understands the Redis protocol.

Think of it like:

    Browser
        │
    HTTP
        ▼
    FastAPI

The browser knows HTTP.

Similarly,

    FastAPI
        │
    Redis Protocol
        ▼
    Redis Server

The redis Python package knows the Redis protocol.

# 4. Step 2 – Create redis_client.py

Let's Understand Every Parameter

# host
host="127.0.0.1" Redis is running on your own laptop.

Later in Docker: host="redis"

Later in AKS: host="azure-cache.redis.cache.windows.net"

Notice how only the address changes.

# port
port=6379

Redis's default listening port.
Exactly like:
      FastAPI → 8000
      SQL Server → 1433
      PostgreSQL → 5432

# db
db=0

Redis has logical databases. Think of them as small namespaces.

For learning, we'll use database 0.

# decode_responses

Without: redis_client.get(...) returns: b"alice" (bytes)

With: decode_responses=True  returns: "alice" (string)

This makes our code easier to read.

# 6. What Happens Internally?

When you visit: GET /redis/ping

The flow is:

      Browser
            │
      HTTP GET
            │
            ▼
      FastAPI
            │
            ▼
      redis_client.ping()
            │
      Redis Protocol
            │
      TCP Socket
            │
            ▼
      Redis Server
            │
      PONG
            │
            ▼
      FastAPI
            │
      JSON
            ▼
      Browser

Notice something beautiful.

FastAPI is now acting exactly like redis-cli. The only difference is:

| redis-cli            | FastAPI                   |
| -------------------- | ------------------------- |
| Human types commands | Python code calls methods |
| Prints to terminal   | Returns JSON              |

Both speak the same Redis protocol.

# 8. Why Didn't We Modify session_store.py Yet?

Because we follow an enterprise migration strategy.

Phase 1:

      Application
      ↓
      Verify Infrastructure

Phase 2:

      Infrastructure Verified
      ↓
      Migrate Business Logic

If something fails now, we know it's a connectivity problem, not a session-storage problem.
