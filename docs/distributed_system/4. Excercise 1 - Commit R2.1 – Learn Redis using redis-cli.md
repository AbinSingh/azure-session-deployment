# 1. Big Picture

Notice something familiar. This is exactly like:

    Browser
        │
     HTTP

except now:

Browser → redis-cli
FastAPI → redis-server

You're learning the protocol directly.

# 2. What is the Redis Protocol?

When you type GET username

Redis doesn't magically understand English.

redis-cli converts your command into the RESP (Redis Serialization Protocol) and sends it over a TCP socket.

So internally:

    redis-cli
         │
    RESP Protocol
         │
    TCP Socket
         │
    redis-server

This is similar to:

    Browser
    ↓
    HTTP
    ↓
    FastAPI

# 3.2 Store Your First Value

Type
    SET username bob
Expected
    OK

# What happened?

Redis memory now contains

username
↓
bob

Think of it as

{
    "username": "bob"
}

except this dictionary belongs to Redis, not Python.

# 3.3 Read the Value

Type
GET username

Expected
"bob"

# Flow
    Client
    ↓
    GET username
    ↓
    Redis
    ↓
    Look up key
    ↓
    Return value

# 3.4 Change the Value

SET username alice
Then
GET username

Result : "alice"

Notice : Redis replaced the previous value.

Exactly like Python: my_dict["username"] = "alice"

# 3.5 Delete the Value
    DEL username

# 3.6 List Keys
Let's create some keys.

SET city Bangalore
SET language Python

Now : KEYS *

Expected: 
1) city
2) language
Why is this useful?
Because now you can literally inspect Redis memory.

# 3.7 Check if a Key Exists
    EXISTS city

# 3.8 Set an Expiration

One of Redis's superpowers is automatic expiration.

Let's create: SET session123 alice

Now : EXPIRE session123 30

Meaning : Delete this key after 30 seconds.

# 3.9 Check remaining lifetime.
    TTL session123

# 3.10 Store JSON

# 3.13 Mapping Redis Commands to Python
Python Dictionary	Redis
d[key] = value	    SET key value
d[key]	            GET key
key in d	        EXISTS key
del d[key]	        DEL key
d.keys()	        KEYS *

Notice:
Redis feels very much like a remote dictionary.


# 3.14 How This Relates to Our FastAPI App

Currently

sessions[session_id] = session

Soon it becomes

redis.set(session_id, json.dumps(session))

Current

sessions.get(session_id)

Becomes

redis.get(session_id)

Current

sessions.pop(session_id)

Becomes

redis.delete(session_id)

See how only the implementation changes, not the rest of the application? That's exactly why we isolated all session operations in session_store.py.


# 4. Mental Model

At this point, I want you to stop thinking:

"Redis is a database."

Instead, think:

    Redis
    
    ↓
    
    Another server process
    
    ↓
    
    Listening on TCP port 6379
    
    ↓
    
    Accepting commands
    
    ↓
    
    Returning responses

This mental model will make the Python integration feel natural, because your FastAPI application will simply become another client, just like redis-cli.
